# Branding Rules

## Overview

The Ivy's Challenge name, logo, visual identity, and project presentation belong to the original creator.

This repository allows community use, contribution, and forking, but it does not transfer ownership of the Ivy's Challenge brand.

## What You May Do

- fork the project
- create modified versions
- describe your work as a community fork based on Ivy's Challenge
- reference the original project for attribution and technical context

## What You Must Not Do

- present a fork as the official Ivy's Challenge project
- imply endorsement, partnership, or approval from the original creator without permission
- reuse the Ivy's Challenge identity for a commercial edition or commercial hosted service
- create product names that suggest an official expansion of the brand

## Naming Guidance

Allowed examples:

- "Community fork based on Ivy's Challenge"
- "A non-commercial fork inspired by Ivy's Challenge"
- "Modified learning app derived from Ivy's Challenge"

Not allowed examples:

- "Official Ivy's Challenge Pro"
- "Ivy's Challenge Enterprise"
- "Ivy's Challenge Cloud"
- "The official next version of Ivy's Challenge"

## Visual Identity

If you publish a modified version, you should replace or clearly distinguish any branding elements that could confuse users, including:

- project name in the app shell
- logo usage
- landing page branding
- product marketing text

## Visual Assets

- Ivy's Challenge visual assets, including avatars, illustrations, and UI graphics, are original project assets.
- These assets are part of the project's visual identity.
- They may be used as part of Ivy's Challenge source code.
- They may not be extracted, redistributed, or sold as standalone commercial assets.
- Modified forks should not present these assets as an independent brand identity.

## Endorsement Rule

Modified versions must not imply official endorsement.

If you want to use the Ivy's Challenge brand in a way that suggests official approval, release partnership, or authorized product status, you must first obtain explicit permission from the original creator.
